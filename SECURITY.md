# Security Policy

This repository publishes security guidance. Incorrect guidance can create insecure systems, so it is treated as a security issue.

## Report A Problem

Open a private security report if you discover:

- guidance that can directly lead to insecure implementations
- broken controls in helper scripts or validation logic
- missing safeguards in examples, automation, or CI checks

## What To Include

- affected file and section
- why the current guidance is unsafe or incomplete
- expected secure behavior
- a minimal reproduction or implementation scenario when possible

## Scope

In scope:

- unsafe security advice
- misleading defaults
- broken validation
- automation bugs that cause the wrong skill or wrong path to be used

Out of scope:

- stylistic wording changes with no security effect
- requests for new frameworks without a concrete problem statement
